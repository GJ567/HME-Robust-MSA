# HME 可改位置分析

## 1. generate_missing_matrix

当前作用：生成缺失模态矩阵 ps，控制每个样本哪些模态缺失。

当前问题：
- 主要是随机缺失
- 没有考虑真实低质量场景
- 没有考虑不同模态质量差异

后续可改：
- 质量驱动缺失
- 同时模拟缺失、噪声、低质量
- 文本 mask、音频加噪、视觉扰动

适合我的方向：高。

## 2. generate_missing_modalities

当前作用：根据 ps 对文本、音频、视觉做缺失处理。

当前问题：
- 缺失处理相对简单
- 不能模拟“模态存在但质量差”的情况

后续可改：
- 模态置零
- 模态加噪声
- 模态降权
- 根据质量分数动态处理模态

适合我的方向：高。

## 3. get_similar_embeddings

当前作用：在当前 batch 内找相似样本，并平均相似样本表示。

当前问题：
- 只在当前 batch 内找，相似样本范围小
- 只用阈值筛选
- 相似样本直接平均，没有区分相似度高低
- 没有考虑样本质量

后续可改：
- top-k 相似样本
- 相似度加权平均
- 质量感知检索
- 情感记忆库检索
- 跨 batch memory bank

适合我的方向：非常高。

## 4. VIB_estimate

当前作用：对 hyper 表示进行信息压缩和不确定性估计。

当前问题：
- 只依赖模型自身估计不确定性
- 没有显式引入模态质量信息

后续可改：
- 加入质量分数辅助不确定性估计
- 做不确定性校准
- 加质量一致性 loss

适合我的方向：中高。

## 5. Attention_Uncertainty_2

当前作用：把单个模态的原始表示和 hyper 表示融合起来。

当前问题：
- 主要依赖不确定性
- 没有显式考虑模态质量

后续可改：
- 加质量感知门控
- 融合权重同时考虑 std 和 quality
- 低质量模态少信，高质量模态多信

适合我的方向：非常高。

## 6. Attention_Uncertainty_3

当前作用：融合三个模态的 hyper 表示，得到 hyper_all。

当前问题：
- 没有显式建模模态质量
- 没有专家路由机制

后续可改：
- 质量感知 hyper 融合
- MoE 专家融合
- 可靠性路由

适合我的方向：中高。

## 7. fusion_all

当前作用：最终 MLP 预测头。

当前问题：
- 融合方式比较简单
- 只是 Linear + Dropout + ReLU + Linear

后续可改：
- MoE 动态融合
- 质量感知专家路由
- 可靠性门控融合

适合我的方向：中高。

## 8. train_epoch 里的 loss

当前作用：主任务 loss + 单模态辅助 loss + info loss。

当前问题：
- 没有质量约束
- 没有检索增强一致性约束
- 没有不确定性校准 loss

后续可改：
- 质量一致性损失
- 不确定性校准损失
- 检索增强对比损失

适合我的方向：中高。
